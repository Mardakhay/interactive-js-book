What is JavaScript?

üî≠ Scope
Where variables live and can be accessed

Global vs Block Scope

const color = 'blue'; // Global scope

const logSkyColor = () => {
  let localColor = 'pink'; // Block scope (local)
  console.log(color);      // Can access global
  console.log(localColor); // Can access local
};

logSkyColor();
// console.log(localColor); // ReferenceError!

Scope Pollution

Too many global variables can cause naming collisions and bugs. Always prefer block scope.

const logSkyColor = () => {
  const dusk = true;
  let color = 'blue';
  if (dusk) {
    let color = 'pink'; // New variable, different scope
    console.log(color); // 'pink'
  }
  console.log(color); // 'blue'
};
üí° Best practice: Use const and let (block scope). Avoid var (function scope, can leak).

üõ°Ô∏è Optional Chaining & Nullish Coalescing
Safe access to deeply nested properties

Optional Chaining ?.

const user = { name: 'Alice', address: { city: 'Paris' } };

// Without ‚Äî throws TypeError if undefined:
// user.address.street.name

// With optional chaining ‚Äî returns undefined safely:
const street = user?.address?.street?.name;
console.log(street); // undefined (no error!)

// With arrays:
const users = null;
console.log(users?.[0]); // undefined

// With functions:
console.log(user.greet?.()); // undefined (no error)

Nullish Coalescing ??

Returns right side only if left is null or undefined (not falsy 0, ''!).

const count = 0;
console.log(count || 10);  // 10 ‚Äî 0 is falsy
console.log(count ?? 10);  // 0  ‚Äî 0 is NOT nullish

const name = null ?? 'Guest'; // 'Guest'

Every object has a hidden link to its prototype. When you access a property, JS looks on the object first, then its prototype, and so on up to null.

üß† Memory Management
How JavaScript allocates and frees memory

Stack vs Heap

StackHeap
Primitive valuesObjects & arrays
References to heap objectsFunctions
Function call framesDynamic size
Limited spaceLarge, flexible

Garbage Collection

JavaScript automatically frees unreachable memory using:

  • Reference Counting ‚Äî tracks how many references point to an object
  • Mark-and-Sweep ‚Äî marks all reachable objects, sweeps the rest
let obj = new Object(); // ref count: 1
let obj2 = obj;         // ref count: 2
obj2 = "string";        // ref count back to 1
obj = null;             // ref count: 0 ‚Üí memory freed

Common Memory Leaks

  • Messy closures ‚Äî closures retaining large objects
  • Dangling timers ‚Äî setInterval not cleared
  • Forgotten event listeners ‚Äî not removed when unneeded
// Prevent timer leak:
const intervalID = setInterval(cb, 1000);
clearInterval(intervalID); // Always clear!
